# 模具：多卡（-m）

## 步骤 1：读取模板

Read `assets/poster_template.html`

如果用户明确说「小红书」「图卡」「卡组」「多卡」「1080x1440」，或调用方来自 `content-illustrate` 的小红书卡组阶段，同时 Read `references/xiaohongshu-card-quality.md`。

## 步骤 1.5：色调感知

普通多卡与长图模具共享同一套色调系统。根据内容气质选择 `{{BG_COLOR}}` 和 `{{ACCENT_COLOR}}`：

| 内容气质 | `{{BG_COLOR}}` | `{{ACCENT_COLOR}}` | 触发信号 |
|----------|---------------|-------------------|----------|
| 思辨/哲学 | `#FAF8F4` | `#7C6853` | 认知、思维、本质、意义、哲学 |
| 技术/工程 | `#F5F7FA` | `#3D5A80` | 架构、模型、算法、系统、代码 |
| 文学/叙事 | `#FBF9F1` | `#6B4E3D` | 故事、人物、写作、文字、诗 |
| 科学/研究 | `#F4F8F6` | `#2D6A4F` | 实验、数据、发现、论文、研究 |
| 默认 | `#FAFAF8` | `#4A4A4A` | 无法归类时 |

### 小红书卡组主题选择

如果是小红书卡组，不要直接套上表默认色，也不要默认暗绿终端。先按 `references/xiaohongshu-card-quality.md` 的主题矩阵判断内容气质：

- CLI / 工程工具 → 可使用暗绿终端风
- AI 产品 / 效率工具 → 暖纸感或浅色产品风
- 研究 / 模型 / 数据 → 克制浅底、数据标签
- 商业 / 趋势 / 创业 → 商业简报风
- 风险 / 安全 / 反思 → 暗红警示风
- 创作 / 内容工具 → 编辑部纸感

选定主题后，在 `prompts.md` 或交付说明里记录：

```markdown
主题：<主题名>
理由：<为什么这篇内容适合该主题，而不是暗绿终端>
配色：背景 <HEX> / 强调色 <HEX>
结构：封面钩子 → 痛点 → 机制 → 能力 → 上手/场景 → 边界
```

## 步骤 2：内容预处理

- 识别标题行（`#`/`##`/`###` 开头，或独立短行）
- 识别引用块（`>` 开头）
- 识别加粗（`**text**`）
- **识别金句**：独立成段的短句（通常 < 25 字），承载核心洞察，用 `.highlight` 渲染
- 按空行分割为段落列表

### 小红书卡组内容改写

小红书卡组不是文章分页。先把文章重写成 5-7 张卡的卡组大纲，再生成 HTML：

1. 封面钩子：痛点 / 反常识 / 明确收益，不只是复述文章标题。
2. 痛点共鸣：解释读者为什么需要这个工具或观点。
3. 机制解释：用流程、对比或结构图说明核心变化。
4. 核心能力：3-4 个能力块，每块要有一句人话解释。
5. 快速上手 / 使用场景：有命令就做步骤；没有命令就做适用场景。
6. 适合谁 + 边界：讲清适用对象、限制和风险。

如果内容不是工具类，按 `references/xiaohongshu-card-quality.md` 的替换规则调整第 5 张，不要硬塞“快速上手”。

## 步骤 3：计算视觉重量

模板在 1080x1440 全分辨率渲染，正文 36px，行高 1.7。

- 普通段落：字符数 × 1.4
- 标题行（h1 首卡 84px）：字符数 × 6.0
- 金句（`.highlight` 40px + 左边框 + 上下留白）：字符数 × 3.0
- `.item` 条目组（label + 正文）：字符数 × 1.8
- 引用块：字符数 × 1.7
- 分割线（divider）：固定 60 权重
- 代码块：字符数 × 2.2
- Running title（续页头部）：固定 70 权重

## 步骤 4：贪心切分

- 阈值：每卡约 **380** 字符等价视觉重量
- 逐段累加，超过阈值时在当前段之前切分
- **切分规则**：
  - 绝不在句子中间切
  - 优先在段落/条目/章节边界切
  - 标题不落单（必须跟至少一个内容元素在同一卡）
  - 超长单段在句号处强制切
  - 一个章节（h2 + 3 items）通常刚好一卡

**特殊情况**：
- 只有一张卡：不显示页码
- 多张卡：显示 `1 / N` 格式页码

小红书卡组优先人工控制每张信息量，不完全依赖贪心切分。每张卡只承载一个问题：

- 封面：一个钩子 + 一个解释句 + 一个核心视觉
- 痛点：3-5 个痛点或一个强对比
- 机制：一个流程图 / 对比图 / 命令链路
- 能力：3-4 个能力块
- 步骤：3-5 步，命令不能尴尬断行
- 边界：适合谁 + 别神化

## 步骤 5：格式化为 HTML

**基础元素：**
- 普通段落 → `<p>文本</p>`
- 章节标题（##/### 级别） → `<h2>标题</h2>`
- 引用 → `<blockquote><p>引用</p></blockquote>`
- 加粗 → `<strong>文本</strong>`
- 列表 → `<ul><li>...</li></ul>`

**金句（独立成段的核心洞察短句，视觉突出）：**
```html
<p class="highlight">金句文本</p>
```
判断标准：独立成段、< 25 字、承载关键洞察。用 `.highlight` 而非 `<p><strong>`。

**条目组（有标题+正文的并列条目）：**
```html
<div class="item">
  <p class="label">条目标题</p>
  <p>条目正文</p>
</div>
```

**副标题标签：**
```html
<p class="subtitle">标签文字</p>
```

**分割线（章节之间）：**
```html
<div class="divider"></div>
```

**小红书卡组增强元素：**

需要时自行在 HTML 中加入轻量组件，不局限于基础段落：

- `topbar`：页码 + 系列名
- `panel`：痛点、能力、步骤卡片
- `flow`：网站 / 工具 / Agent 等链路
- `terminal`：命令行示例
- `steps`：快速上手步骤
- `quote`：一句收束判断

这些组件应服务信息表达，不要为了装饰而堆组件。

## 步骤 6：渲染模板

对每张卡片，替换模板变量：

| 变量 | 规则 |
|------|------|
| `{{BG_COLOR}}` | 步骤 1.5 确定的背景底色 |
| `{{ACCENT_COLOR}}` | 步骤 1.5 确定的强调色 |
| `{{HEADER_BLOCK}}` | 续页卡：`<div class="header"><span class="running-title">文章标题</span></div>`；首卡或单卡：空字符串 |
| `{{TITLE_BLOCK}}` | 首卡有标题时：`<div class="title-area"><h1>标题</h1></div>`；续页卡或无标题时：空字符串 |
| `{{BODY_HTML}}` | 步骤 5 生成的 HTML |
| `{{SOURCE_LINE}}` | 内容来源（可选）：`<span class="info-source">来源文字</span>`，无来源时空字符串 |
| `{{PAGE_INFO}}` | 多卡时 `1 / 3`，单卡时空字符串 |

**结尾标记**：仅在最后一张卡的 `{{BODY_HTML}}` 末尾追加 `<p style="text-align:right;font-size:16px;color:#ACACB0;margin-top:40px;">∎</p>`。非末页不加。

写入：`/tmp/ljg_cast_poster_{name}_{N}.html`

小红书卡组如果使用了自定义主题和增强组件，可以生成一份单页多卡预览 HTML，支持通过 query 参数或重复 section 分卡截图。必须把源 HTML 或核心 prompt 留底，方便复现和二次修改。

## 步骤 7：截图

```bash
node assets/capture.js /tmp/ljg_cast_poster_{name}_{N}.html ~/Downloads/{name}_{N}.png 1080 1440
```

多张卡片可并行截图。

截图后打开至少首图、机制图、步骤图、最后一图检查：

- 标题是否被不自然截断
- 命令 / 英文是否断行
- 正文是否过小或过密
- 主题是否符合内容，而不是默认暗绿终端
- 是否有一张能讲清“它怎么工作”的结构图

交付时报告卡片数量 + 每张摘要（前 30 字）。
