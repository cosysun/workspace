# 公众号正文配图规划

用于给公众号文章生成正文配图时，先判断每张图的任务，再选择 AI / HTML / SVG 路径。目标是提高文章质感，同时保留信息准确性和成本控制。

## 1. 每张图必须有任务

正文配图不是按 H2 机械插图。生成前先给每张图定义任务：

- **建立记忆点**：用概念图、场景图、氛围图让读者记住主题。
- **解释机制**：用流程图、架构图、对比图说明复杂关系。
- **降低阅读阻力**：用步骤图、清单图把长段落变轻。
- **补充证据**：用数据图、表格、截图支撑判断。
- **节奏休息**：在长文中提供视觉停顿，但不能只是装饰。

如果一张图说不清任务，就不要生成。

## 2. 路径选择

| 图型 | 优先路径 | 适用场景 | 注意 |
|---|---|---|---|
| 封面图 | AI | 决定点击率，需要质感和视觉锚点 | 核心文字放中央安全区 |
| 概念 / 氛围图 | AI | 文章开头、观点转折、工具定位 | 不放长文字，避免中文渲染错误 |
| 机制 / 架构图 | SVG 或 HTML | 流程、系统关系、组件连接 | 文字必须准确，可用 `baoyu-diagram` |
| 步骤 / 清单图 | HTML | 安装步骤、使用流程、对比清单 | 适合代码控制和二次修改 |
| 数据 / 表格图 | HTML | 多数字、多字段、精确表格 | 不用 AI 生成精确文字 |
| 产品 / 网页截图 | 现成图或本地截图 | 真实界面、项目页面、功能展示 | 要确认版权和清晰度 |

## 3. 默认组合

按文章长度和内容密度规划：

- **短文（<1500 字）**：封面 + 1 张机制/场景图。
- **中等文章（1500-3000 字）**：封面 + 2-3 张正文图。
- **长文（3000 字以上）**：封面 + 3-5 张正文图，最多 1 张纯氛围图，其余服务理解。

AI 成本默认只保证封面。正文如果要加 AI 概念图，必须说明为什么 HTML / SVG 不能替代，并计入预算。

## 4. 内容感知风格

正文图的风格要和文章气质匹配，不要所有科技文都用暗色终端：

- AI 工具 / 效率：暖灰、浅色产品卡、低饱和橙蓝。
- CLI / 工程工具：暗色终端、网格、等宽字体。
- 研究 / 数据：浅灰绿、数据标签、克制网格。
- 商业 / 趋势：象牙白、深蓝或金色点缀、简报结构。
- 风险 / 安全：炭灰、暗红、警示标签。
- 创作 / 内容：纸感、编辑部排版、暖棕点缀。

同一篇文章内，封面和正文图要共享一个色彩系统，但正文图可以更克制，避免抢正文。

## 5. `prompts.md` 留底格式

每张正文图都要记录目的和生成依据：

```markdown
### image-N.png

- 位置：<放在哪个章节后>
- 任务：<建立记忆点 / 解释机制 / 降低阅读阻力 / 补充证据 / 节奏休息>
- 图型：<概念图 / 机制图 / 步骤图 / 数据图 / 截图>
- 路径：<AI / HTML / SVG / 现成图>
- 风格：<主题与配色>
- 内容摘要：<这张图讲什么>
- 源文件或 Prompt：<HTML/SVG 路径，或完整 prompt>
```

## 6. 出图自检

- [ ] 这张图是否有明确任务？
- [ ] 图的位置是否服务阅读节奏？
- [ ] 文字是否准确、清晰、没有过小？
- [ ] 图型路径是否合理，精确文字没有交给 AI 硬画？
- [ ] 风格是否和文章气质一致？
- [ ] 和小红书卡组是否重复过多？
- [ ] 是否已在 `prompts.md` 留底？
