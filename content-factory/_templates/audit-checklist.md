# 三遍审校 Checklist

文章 slug：
起始版本：v1  →  v2（内容） →  v3（风格/降AI味） →  final（细节）

## 第一遍：内容审校（v1 → v2）

### 事实准确性
- [ ] 数据/百分比/时间准确
- [ ] 产品名/公司名/人名正确
- [ ] 技术术语使用正确
- [ ] 所有外链有效
- [ ] 引用来源标注清楚

### 逻辑清晰性
- [ ] 前后论述无矛盾
- [ ] 推理过程合理
- [ ] 论据支撑观点

### 结构合理性
- [ ] 开头引入自然
- [ ] 各部分过渡流畅
- [ ] 没有跑题
- [ ] 结尾有收束

### 真实性
- [ ] 所有案例真实可验证
- [ ] 无编造数据

## 第二遍：风格审校（v2 → v3）—— humanizer + huashu-proofreading 跑

### 删套话
- [ ] "在当今时代" / "在...大背景下"
- [ ] "综上所述" / "总而言之"
- [ ] "值得注意的是" / "需要强调的是"
- [ ] "随着...的发展"

### 拆 AI 句式
- [ ] "不仅...而且..." 堆叠
- [ ] "一方面...另一方面..."
- [ ] "既...又..."
- [ ] "不是...而是..." 连续 3 次以上

### 换书面词为口语
- [ ] "显著提升" → 具体数字
- [ ] "充分利用" → "用好"
- [ ] "进行操作" → 直接动词（点击/输入）
- [ ] "实现功能" → "做到"

### 去列表癖
- [ ] "首先/其次/最后" 过度使用
- [ ] 能散文说清的不列表
- [ ] 小标题不密集（非每 200 字一个）

### 去 -ing 伪深度（humanizer 重点）
- [ ] highlighting/underscoring/emphasizing/symbolizing/reflecting 过度使用

### 去 em dash 滥用（humanizer 重点）
- [ ] 每段 em dash 不超过 1 个

## 第二遍半：流畅度审校（v3 → final 之前必跑）

> **总原则**：让读者一遍读懂 > 字数压在下限。
> 写完每段问自己：「如果我是第一次读这篇文章的人，读到这里有没有任何一刻需要回读上一句才能理解？」有就改。
> Brief 写的字数是**上限**，不是目标——为流畅度可以让字数下限。

### 1. 倒序叙事
- [ ] 没有先给"正确版本"再说"我之前错了"
- [ ] 任务句只交代抽象目标，具体数字/参数留到"修正后"那一句出现
- [ ] 按「意图 → 试错 → 修正 → 结果」的自然时序

### 2. 砍 referent / 抽象比喻
- [ ] "再压一遍，**干净**" 类——干净指什么写清楚（例："5 个场景这次都没误触发"）
- [ ] "再筛一遍" 类——筛什么对象、刷掉什么标准写清楚
- [ ] 形容词（清晰、干净、漂亮、达标）后面有具体定义
- [ ] 抽象比喻（点头、过关、上岸、出圈）改成视觉化动作（**打钩 / 通过 / 三个都答"是"**）
- [ ] 读者脑海能看到一个具体画面

### 3. 主宾倒置 / 抽象语序
- [ ] 默认主动语序「主语-动词-宾语」
- [ ] 没有"把 X 让 Y 怎么样"这种主宾绕行结构
- [ ] 复杂概念有具象动词承担（"填上"比"填掉"更具象）

### 4. 看似精确的数字代替自然表达
- [ ] 没有"8/10 的场景" → 用"80%"或"大部分"
- [ ] 不在散文里用分数符号（除非真在做实验报告）

### 5. 砍连接词当冗余
- [ ] 行动→后果之间有 **结果 / 于是 / 这样 / 这下**
- [ ] 顺承思路有 **顺着这个思路 / 接下来 / 那么**
- [ ] 转折有 **不过 / 但 / 反过来**
- [ ] 强调有 **关键是 / 核心在于**

### 6. 主语缺失
- [ ] 除非主语在前句明确（连续动作），否则补主语
- [ ] "我"和"我们"该写就写，不省

### 7. 段落超载
- [ ] 一段不超过 3 句话（公众号手机阅读上限）
- [ ] 一段不塞 3+ 件事（机制 / 例子 / 取舍各自一段）
- [ ] H2/H3 标题后第一段保持短开场（1-2 句）

### 8. 列表机会未抓
- [ ] 4+ 并列项不堆在一段，改用 bullet
- [ ] 多个 case 并置时，bullet 化让"数得清"
- [ ] 列表化让数量与下文呼应（"四个角度，干的是同一件事"前面正好是 4 bullet）

## 第三遍：细节打磨（v3 → final）

### 强开头
- [ ] 第一句让人想读下一句
- [ ] 不铺垫、不背景、不"自古以来"

### 句子节奏
- [ ] 长短错落（不全长或全短）
- [ ] 同一种句式全文最多一次

### 多巴胺密度
- [ ] 每 3-5 段读者有一次"原来如此"

### 微幽默
- [ ] 有 1-2 处微幽默（不做作）

### 概念把手
- [ ] 复杂概念有类比或具体例子

### 格式
- [ ] 标点规范
- [ ] 无错字
- [ ] 加粗/斜体不滥用

### 字数（3.4.5 收口）
- [ ] 中文字符数在 brief 设定范围内（或 ≤ 上限 +20% 软超且已记录原因）
- [ ] 没有为了凑字数灌水，也没有为了砍字数损伤事实/逻辑
