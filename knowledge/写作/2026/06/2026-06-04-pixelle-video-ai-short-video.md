你可能经历过这种场景：脑子里有个选题，但一想到要写文案、找配图、录配音、剪视频，五个步骤折腾两小时，最后只产出一支 5 分钟的视频。

如果有个工具，你只输入一句话，它帮你把文案、配图、配音、BGM、合成全搞定呢？

今天介绍一个开源项目：**Pixelle-Video**。

## 一句话出片是什么体验

Pixelle-Video 是阿里 AIDC-AI 团队开源的 AI 短视频引擎。GitHub 上 21.3k Stars，Apache 2.0 协议。

你只需要在输入框写一句话，比如"为什么我们还没找到外星文明"，它会自动完成四件事：

1. LLM 写解说词，按分镜拆段
2. 每段配一张 AI 生成的图（或视频片段）
3. TTS 引擎把文字转成语音
4. 加上 BGM，合成最终视频

从输入到出片，几分钟。你不用写文案，不用找图，不用录音，不用打开剪映。

它不是 Sora 那种"输入文字生成一段视频"的模型。它是一条**完整的视频生产线**，把文案、画面、声音、字幕串起来，端到端自动跑完。

实际操作就三步：左侧栏选"AI 生成内容"，输入主题；中间栏选 TTS 和画面模板；右侧栏点"生成视频"。进度条实时显示当前跑到哪一步，比如"分镜 3/5 - 生成插图"。跑完后视频自动播放，同时保存到 output 文件夹。

## 它跟同类工具有什么不同

说到 AI 短视频工具，你可能听过 MoneyPrinterTurbo（20k+ Stars）。两者看起来很像，但关键区别在"画面从哪来"。

MoneyPrinterTurbo 从 Pexels 素材库搜图。通用话题还行，但碰到小众主题（比如"资治通鉴解读"），素材库找不到匹配的画面，要么硬凑要么重做。实测小众主题的重编辑率达到 **61%**。

Pixelle-Video 用 ComfyUI 逐帧生成画面。每个分镜的图都是 AI 画的，风格统一，不存在"找不到素材"的问题。同条件下重编辑率只有 **24%**。

最核心的差异就一句：一个从素材库里挑，一个从零画出来。

还有几个 MoneyPrinterTurbo 没有的扩展模块：

- **数字人口播**：TTS 语音 + 数字人形象，生成面对镜头说话的视频，支持多语言
- **动作迁移**：上传参考视频，把动作"搬"到你的图片上，比如让你的猫跟着参考视频跳舞
- **图生视频**：静态图变成动态片段，适合做开头或产品展示

这些扩展模块都是 2026 年新增的，项目还在快速迭代。6 月 1 日的更新刚加了直连 API 媒体模型配置，不用再依赖 ComfyUI 工作流，直接在 Web 界面填 API Key 就能调 DashScope、Seedream、Kling 这些服务。

声音方面，除了免费的 edge-tts，还支持 **Index-TTS 声音克隆**：上传一段参考音频，AI 就能模仿这个人的音色来配音。配合数字人口播模块，可以做"特定人物口播"的效果，这在开源短视频引擎里比较少见。

## 三条流水线，三种创作模式

Pixelle-Video 内置三条流水线，适配不同场景：

**Standard 流水线**：最常用。输入主题，AI 从零创作。适合"我有个想法但还没写稿"的时候。

**Custom 流水线**：用 HTML 模板精细控制每一帧的呈现方式。适合有品牌视觉要求的频道，或者你就是想要某个特定的排版风格。

**Asset Based 流水线**：你已经有了照片或视频素材，上传后 AI 分析内容，自动生成匹配的脚本。适合"我拍了素材但不知道怎么串起来"的情况。

三条流水线共享同一套底层能力。你可以随时换 LLM（GPT / 通义千问 / DeepSeek / Ollama），换 TTS（edge-tts / Index-TTS + 声音克隆），换画面模型（FLUX / SDXL / WAN 2.1），每个环节独立可替换。

这种"原子能力灵活组合"的设计，是它区别于同类工具的另一个优势。你不喜欢默认的配图模型？换成 SDXL。觉得通义千问的文案太规矩？换 DeepSeek 试试。每个环节都是即插即用的。

## 怎么装？两种方式

### Windows 用户：一键整合包

下载整合包，解压，双击 `start.bat`。浏览器自动打开，开箱即用。不用装 Python、不用装 ffmpeg。

### macOS / Linux 用户：源码安装

需要两个前置：`uv`（Python 包管理器）和 `ffmpeg`。装好后三条命令：

```bash
git clone https://github.com/AIDC-AI/Pixelle-Video.git
cd Pixelle-Video
uv run streamlit run web/app.py
```

`uv run` 会自动安装所有依赖。

启动后在 Web 界面的"系统配置"里填 API Key：LLM 至少配一个（推荐通义千问，便宜），图片生成配 ComfyUI 地址或直连 API Key。

## 免费方案怎么搭

Pixelle-Video 支持完全免费运行：

- LLM 用 **Ollama**，本地跑，不要 API Key
- 图片生成用 **ComfyUI 本地部署**，用开源模型（FLUX 等），不花钱
- TTS 用 **edge-tts**，微软免费服务
- BGM 用内置音乐

代价是你需要一张还行的显卡（测试环境 RTX 4070）。没有显卡的话，直连 API 方案单条视频成本也很低，通义千问的 LLM 调用几分钱，图片生成几分钱，一条视频几毛钱以内。

## 实话实说：它还不够完美

几个短板：

**ComfyUI 配置门槛**。从没用过 ComfyUI 的话，第一次配置大概需要 90 分钟。Windows 整合包缓解了这个问题，但出错排查还是得懂 ComfyUI 节点逻辑。

**依赖链重**。25+ 依赖包，moviepy 锁定在 1.0.3 版本，Python 要求 ≥3.11。环境不够干净的时候容易撞依赖冲突。

**没有实时预览**。流水线全跑完才能看到最终视频。中间某个环节效果不对，只能改参数重跑整条。

**视频合成能力有限**。用的是 moviepy + ffmpeg，转场和特效比专业剪辑软件弱很多。它算"初稿生成器"，不是"成品发布器"，如果你对最终效果有要求，还是得进剪映精修。

## 谁适合用

- 做知识科普、人文纪实、小说解说类短视频的创作者，素材库找不到图，Pixelle-Video 直接画
- 想要统一视觉风格的品牌频道，AI 生图风格一致，不会出现素材拼凑感
- 预算有限的个人创作者，免费方案真的能跑
- ComfyUI 老用户，直接接入你已有的工作流

如果你的内容偏通用（健康科普、理财清单这类），MoneyPrinterTurbo 可能更省事。但一旦你的主题比较小众，或者你对画面风格有要求，Pixelle-Video 的优势就很明显了。
